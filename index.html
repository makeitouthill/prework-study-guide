HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Best practice when creating the body of a webpage is to use semantic HTML elements whenever possible.
    To allow the webpage accessible to the widest varity of users agents.
  • Semantic HTML elements to defines the role of their content, as well as to give structure to the content of the webpage.
  • "Parent/Child" relations refer to elements that are nested inside a parent element
  • "Element" and "Tag" are used interchangeably by developers,
    but element is more specific in describing the entire opening and closing tag
    where as tag is just refering what's in the bracket
  • Heading tags h1-6 are used for creating titles and subtitles to help you organize a page
  • Order matters when creating headings use "h" tags in order from 1-6
    to make information accessible to users with screen readers. h1 is biggest and should only be used once per page!
  • "p" elements are paragraph containors or blocks of text
  • Unorder list "ul", Order list "ol", and List items "li"
    create bulleted or numbered list
  • "Src" attribute defines the location, or source of a image file
    and is the most important element in a parent img containor
  • "Alt" attribute contains a text string that describes the image or function
    Alt text can also improve search engine ranking for your website
  • "br" elements creates a line of empty space, or line break between content.
  • Creating links on a page is done by coding in an anchor "a" element
    and including "href" element that going to be the location the link will send you to.
  • Header different than head is something the user is suppose to see and use that goes in the body parent element.
    Header is the section at the top of the page that can provide covers, titles, and navigation
  • Footer elements appear at the bottom of the page.
    tend to have the author, copyright, contact, sitemap, and navigation.
  • Section elements are semantic elements that define section in a document that contains thematic content

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • "git fetch" is downloads but doesn't make changes to your local device,
    "git pull" downloads and changes files on your local device
  • "git push" uploads local repository content to a remote repository
  • git add moves changes to the staging area, gives opportunity to prepare the snapshot before
    commiting it to the official history, so you don't have commit each file individualy
  • git clone creates a copy of an existing Git repository
  • git commit take the staged snapshot and commits to the project history. Combined with git add,
    this defines the basic workflow for all Git users
  • git commit --ammend, this let's you amend the most recent commit. Useful when you forget to stage or omit
    important information from the commit comment.
  • git rebase lets you move branches around, which help avoid unnecessary merge commits
